# Inquiry App Prototype  


## 概要
本リポジトリは、インターンシップ内で行った「英語＊探求学習アプリ」プロジェクトにおいて構想した  
**探究学習 × 英語教育 ** のコンセプトを、  短期間で動作するプロトタイプ（PoC）として実装したもの**です。

英語を「暗記する対象」ではなく、  
考えるためのツールとして使う学習体験を提供することを目的としています。

---

##  実習期間内に行ったこと
本プロジェクトは、約1か月間の実習期間を通して取り組んだものであり、  
プログラミング作業だけでなく、企画・検討段階から段階的に進めました。

実習期間の前半では、英語学習および探究学習に関する課題、既存サービス、事例について調査・整理を行い、  
「英語を知識として学ぶのではなく、思考を深めるための手段として活用する」という方向性を設定しました。  
その上で、写真を起点とした探究活動から英文生成、クイズ、振り返りへとつながる学習フローを設計しました。

後半では、AIの活用方法やプロンプト構成について検討を行い、  
実習期間内で実装可能な範囲に機能を絞った上で、  
プロトタイプとして必要最低限の機能要件を定義しました。

実際のアプリ実装については、 設計した学習フローをもとに、動作するPoC（概念実証）として完成させることを目標に取り組みました。
---

## アプリの特徴
- 写真やテーマを起点にした探究型学習フロー
- AIによる英文ストーリー生成
- 自動生成クイズと即時フィードバック
- 学習進行に応じた段階的な問題提示
- プロンプト設計を外部ファイル化し、調整しやすい構成

---

## 実装内容
-## 実装内容（API利用を含む）

本プロトタイプでは、GooglevisionAPI、GeminiAPIの2つのAPIを利用し、学習フローの中核となる  
英文生成およびクイズ生成を行っています。

実装にあたっては、APIの利用回数や応答時間を考慮し、  
1問ずつ個別に生成するのではなく、ストーリー生成と複数問クイズをまとめて生成する設計としました。  
これにより、API呼び出し回数を抑えつつ、学習体験の連続性を維持しています。



APIから取得した生成結果はアプリ内で状態管理を行い、  
クイズ進行、回答判定、フィードバック表示と連動させることで、  
対話的な学習体験として一連の流れが完結するよう実装しています。
 プロンプト設計の分離管理（prompt_*.txt）

※ 本リポジトリは PoC を目的としており、
UIやセキュリティ、モデル最適化は最小構成です。

- 企画（上流）と実装（下流）をつなぐ経験
